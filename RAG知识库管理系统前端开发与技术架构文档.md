# RAG + 知识库管理系统前端开发与技术架构文档

## 1. 项目概述

本项目定位为农业 AI 研究与生产服务平台前端系统，面向农业科研人员、农技专家、种植管理人员与平台运营人员，提供基于 RAG（Retrieval-Augmented Generation，检索增强生成）的知识问答、知识库管理、专家工作台、基因信息查询、病害识别、温室监控与农业数据大屏能力。

系统整体视觉参考提供的界面效果图，采用深色科技风、农业蓝绿色高亮、卡片式数据展示、发光描边、玻璃拟态面板和大屏化信息布局。登录后主界面采用左侧固定侧边栏 + 顶部导航 + 主内容区域的后台系统布局。

前端技术栈：

- React 18
- Vite
- TypeScript
- Ant Design
- Zustand
- React Router
- ECharts / Ant Design Charts
- Three.js / React Three Fiber（用于温室监控 3D 大棚）
- Axios
- ESLint / Prettier

## 2. 建设目标

1. 建设统一的农业 AI 平台门户，支持登录、权限识别与模块化访问。
2. 建设 RAG 问答前端能力，支持基于知识库、文档、专家库、病害库、基因库的智能问答。
3. 建设知识库管理与检索能力，支持文档上传、解析状态、向量化状态、知识分类、问答引用溯源。
4. 建设农业业务模块，包括数据总览、工作台、AI 对话、基因智查、病害识别、温室监控。
5. 建设可扩展的前端工程架构，便于后续接入更多模型服务、知识源、IoT 设备与大屏组件。

## 4. 系统整体页面结构

系统主体包含：

1. 首页 / 登录页
2. 登录后主应用框架
3. 六个一级功能模块，左侧侧边栏顺序固定为：

| 顺序 | 模块 | 页面定位 |
| --- | --- | --- |
| 1 | 数据总览（数据大屏） | 平台核心指标、问答趋势、知识库统计、病虫害预警、地图分布 |
| 2 | 工作台 | 任务待办、知识处理进度、专家审核、常用入口 |
| 3 | AI 对话 | 农业知识 RAG 问答、上下文对话、文档引用、会话管理 |
| 4 | 基因智查 | 基因知识检索、基因功能解释、表达模式分析、文献关联 |
| 5 | 病害识别 | 图片上传识别、病害诊断、防治建议、历史记录 |
| 6 | 温室监控（有 3D 大棚） | 3D 温室展示、环境指标、传感器数据、趋势图和设备状态 |

## 5. 信息架构与路由设计

建议使用 React Router 进行路由管理。

| 路由 | 页面 | 说明 |

| `/login` | 登录页 | 平台登录、验证码、记住登录 |

| `/workspace` | 工作台 | 个人任务、系统任务、快捷入口 |

## 6. 页面功能设计

### 6.1 首页 / 登录页

页面定位：系统入口，突出农业 AI 研究平台品牌形象。

核心内容：

- 平台名称：Yago Intellect 农业 AI 研究平台 / 可按项目实际品牌调整。
- 登录表单：账号、密码、验证码、记住我、登录按钮。
- 登录方式：账号密码登录，可预留手机号、统一认证、微信扫码登录入口。
- 背景风格：农业科技、温室、基因、山地农田等视觉素材，使用暗色遮罩。
- 登录成功后写入 token、用户信息、权限菜单，并跳转数据总览。

交互要求：

- 表单字段必填校验。
- 登录失败展示错误提示。
- 登录中按钮 loading。
- token 过期自动回到登录页。

### 6.2 数据总览（数据大屏）

页面定位：登录后的默认首页，用于展示平台核心运营指标与农业态势。

参考图功能：

- 顶部指标卡片：
  - 实时问答总量
  - 活跃专家智库
  - 知识库文档数量
  - 助农覆盖农户数量
- 趋势图：
  - 亚热带果蔬问答趋势
  - 按 7 天、30 天、90 天切换
- 病虫害暴发预警：
  - 预警名称、地区、风险等级、更新时间
- 知识图谱领域分布：
  - 果蔬育种、病害防治、栽培技术、基因测序等分类占比
- 服务覆盖地图：
  - 广西、广东、云南、福建、海南等地区覆盖热力
- 品种贡献排行：
  - 品种名称、问答贡献量、趋势变化

前端实现：

- 指标卡使用 Ant Design Card + 自定义暗色主题。
- 折线图、环图、地图、排行图使用 ECharts。
- 数据接口支持定时刷新。
- 大屏布局适配 1920x1080，同时支持普通后台响应式布局。

### 6.3 工作台

页面定位：用户登录后的个人操作中心。

核心功能：

- 今日待办：
  - 待审核知识文档
  - 待复核病害识别结果
  - 待回复用户问题
  - 待处理设备告警
- 知识处理进度：
  - 文档上传数量
  - 解析中数量
  - 向量化完成数量
  - 失败任务数量
- 快捷入口：
  - 新建知识库
  - 上传文档
  - 发起 AI 问答
  - 进入病害识别
  - 查看温室状态
- 最近访问：
  - 最近会话
  - 最近文档
  - 最近识别记录
- 系统通知：
  - 知识库更新通知
  - 模型服务状态
  - 设备告警

### 6.4 AI 对话

页面定位：农业知识 RAG 问答入口。

参考图功能：

- 中央欢迎卡片，展示用户名称和 AI 助手定位。
- 推荐问题卡片：
  - 芒果 / 荔枝等亚热带果蔬栽培技术
  - 病虫害防治建议
  - 惠农政策查询
  - 品种选择建议
- 底部输入框：
  - 文本输入
  - 附件上传
  - 发送按钮
- 对话区：
  - 用户消息
  - AI 回答
  - 引用文档
  - 置信度 / 来源
  - 重新生成、复制、点赞、点踩
- 会话管理：
  - 新建会话
  - 历史会话
  - 会话重命名
  - 删除会话

RAG 交互流程：

1. 用户输入问题。
2. 前端发送问题、会话 ID、知识库范围、模型参数到后端。
3. 后端执行 query rewrite、向量检索、关键词检索、rerank、上下文拼接、模型生成。
4. 前端以流式响应方式展示回答。
5. 回答结束后展示引用来源、相关问题与反馈操作。

### 6.5 基因智查

页面定位：面向科研人员的基因知识查询与分析助手。

参考图功能：

- 顶部欢迎卡片：基因智查 AI 助手。
- 推荐任务：
  - 描述芒果 MiCHS 基因功能
  - 推荐细胞代谢相关基因
  - 荔枝 LcMYB1 基因功能
  - 推荐抗旱相关候选基因
- 输入区：
  - 基因名称、物种、表达组织、功能关键词输入
  - 支持自然语言提问
- 结果展示：
  - 基因基础信息
  - 功能注释
  - 表达模式
  - 代谢通路
  - 关联文献
  - 相关基因推荐
- 可扩展子功能：
  - 基因浏览器
  - 序列分析
  - 数据库
  - 文献检索
  - 工具集

前端重点：

- 查询结果使用分区卡片展示。
- 文献引用支持展开、跳转、复制。
- 图表支持表达量热力图、通路关系图、基因关联网络。

### 6.6 病害识别

页面定位：通过图像识别果蔬病虫害，并生成防治建议。

参考图功能：

- 页面标题：亚热带果蔬病害智能视觉识别。
- 上传区域：
  - 点击上传
  - 拖拽上传
  - JPG / PNG
  - 单张大小限制
- 拍摄建议：
  - 对焦清晰
  - 局部特写
  - 避免远景
- 识别结果：
  - 病害名称
  - 置信度
  - 症状描述
  - 发生原因
  - 防治建议
  - 推荐药剂或农艺措施
  - 相似病害对比
- 支持识别的病害类型：
  - 赤腐病
  - 黑穗病
  - 锈病
  - 花叶病
  - 黄叶病
  - 虫害
- 历史记录：
  - 图片缩略图
  - 识别时间
  - 识别结果
  - 复核状态

前端重点：

- 上传组件使用 Ant Design Upload。
- 识别中展示 loading 和进度。
- 图片预览支持放大、删除、重新上传。
- 诊断建议需提示“仅供参考，不构成诊断依据”。

### 6.7 温室监控（有 3D 大棚）

页面定位：展示温室环境、传感器状态、3D 大棚和设备数据。

参考图功能：

- 顶部 3D 大棚区域：
  - 使用 Three.js / React Three Fiber 渲染温室模型。
  - 支持旋转、缩放、视角切换。
  - 大棚关键点显示传感器标签。
- 环境指标：
  - 光合有效辐射
  - 总辐射
  - CO2 浓度
  - 空气温度
  - 空气湿度
  - 大气压
- 实时环境与基质指标：
  - 土温 1-4
  - 土湿 1-4
  - 光照强度
  - 风速
  - 风向
- 气象与农时中枢：
  - 当前时间
  - 日期
  - 农历
  - 节气
  - 室外温度
  - 风力风向
  - 气象坐标
- 趋势图：
  - 24 小时温湿度趋势
  - 可扩展 CO2、光照、土壤温湿度趋势
- 设备状态：
  - 风机
  - 水泵
  - 遮阳帘
  - 喷淋
  - 传感器在线状态

前端重点：

- 3D 场景单独封装为 `GreenhouseScene`。
- 传感器数据使用 WebSocket 或轮询刷新。
- 3D 标注与指标卡联动，点击传感器点位可高亮对应数据卡。
- 移动端可降级为静态大棚图 + 指标列表。

## 7. RAG 与知识库管理前端设计

虽然左侧一级菜单只保留六个主功能，但知识库管理能力应贯穿 AI 对话、基因智查、工作台和数据总览。

### 7.1 知识库核心对象

| 对象 | 说明 |
| --- | --- |
| KnowledgeBase | 知识库，如农业政策库、病害防治库、基因文献库 |
| Document | 文档，如 PDF、Word、Excel、网页、文本 |
| Chunk | 文档切片 |
| EmbeddingTask | 向量化任务 |
| RetrievalResult | 检索结果 |
| Citation | 回答引用来源 |
| Conversation | AI 会话 |
| Message | 对话消息 |

### 7.2 文档上传与处理状态

建议前端在工作台或弹窗中提供知识上传能力：

1. 选择知识库。
2. 上传文件。
3. 展示解析状态：待解析、解析中、解析成功、解析失败。
4. 展示切片状态：待切片、切片中、完成。
5. 展示向量化状态：待向量化、向量化中、完成、失败。
6. 完成后可进入 AI 对话中被检索引用。

### 7.3 问答引用展示

AI 回答下方应展示：

- 引用文档名称
- 片段摘要
- 页码 / 段落位置
- 相似度分数
- 知识库名称
- 查看原文按钮

### 8.2 分层说明

### 8.3 状态管理

使用 Zustand 管理轻量全局状态。

建议拆分：

- `auth.store.ts`：token、用户信息、权限、登录状态。
- `app.store.ts`：主题、侧边栏折叠、全局 loading、面包屑。
- `chat.store.ts`：当前会话、消息列表、流式回答状态。
- `greenhouse.store.ts`：当前温室、传感器数据、选中设备。

服务端数据建议结合接口请求和组件局部状态管理；如项目规模扩大，可引入 TanStack Query 管理缓存、重试和失效更新。

### 8.4 UI 组件规范

基于 Ant Design 二次封装：

- `AppLayout`：登录后整体布局。
- `SidebarMenu`：左侧菜单。
- `TopBar`：顶部导航、搜索、通知、用户信息。
- `MetricCard`：指标卡。
- `GlowCard`：发光科技卡片。
- `ChartPanel`：图表容器。
- `ChatInput`：AI 输入框。
- `CitationList`：引用来源列表。
- `UploadPanel`：图片 / 文档上传。
- `GreenhouseScene`：3D 大棚。

设计要求：

- 主色：科技蓝。
- 辅色：农业绿。
- 背景：深色渐变或深色图片遮罩。
- 卡片：半透明深色背景、1px 蓝色描边、轻微发光。
- 字体：中文优先使用系统字体，数字指标可使用等宽或科技感字体。
- 交互状态：hover、active、selected、loading、disabled 状态清晰。

```

{
  "username": "admin",

```

响应：

```json
{
  "token": "jwt-token",
  "user": {
    "id": "u001",
    "name": "张农业",
    "role": "researcher",
    "avatar": ""
  },
  "permissions": ["dashboard:view", "chat:use"]
}
```

### 9.2 RAG 对话

```ts
POST /api/rag/chat
```

请求：

```json
{
  "conversationId": "c001",
  "question": "芒果炭疽病怎么防治？",
  "knowledgeBaseIds": ["kb_disease", "kb_policy"],
  "stream": true
}
```

响应建议使用 SSE 或 WebSocket 流式返回：

```json
{
  "type": "answer_delta",
  "content": "建议从清园、修剪、药剂防治等方面..."
}
```

最终返回引用：

```json
{
  "type": "done",
  "citations": [
    {
      "documentId": "doc001",
      "title": "芒果主要病害防治技术手册.pdf",
      "page": 12,
      "score": 0.86,
      "snippet": "炭疽病高发于高温高湿环境..."
    }
  ]
}
```

### 9.3 病害识别

POST /api/disease/recognize

响应：

{

## 10. 权限与安全

前端需实现：

- token 存储与自动携带。
- 路由守卫。
- 菜单权限过滤。
- 接口 401 自动退出登录。
- 403 显示无权限页面。
- 上传文件类型和大小校验。
- AI 生成内容免责声明。
- 敏感操作二次确认。

## 11. 性能优化

1. 路由级懒加载，减少首屏包体积。
2. 图表组件按需加载。
3. Three.js 3D 大棚独立 chunk，进入温室监控页面时再加载。

## 12. 开发规范

### 12.1 TypeScript

- 所有接口响应定义类型。
- 避免使用 `any`。
- 业务实体统一放在 `src/types` 或模块内 `types.ts`。
- 表单数据和接口数据分离定义。

### 12.2 组件规范

- 页面组件放在 `features/{module}/pages`。
- 业务组件放在 `features/{module}/components`。
- 通用组件放在 `components`。
- 单个组件尽量控制在 300 行以内。
- 复杂逻辑抽离为 hooks。

### 12.3 样式规范

- 使用 Ant Design 主题 token 管理颜色和圆角。
- 全局布局变量集中维护。
- 页面级样式与组件样式分离。
- 避免大量内联样式。

### 12.4 Git 提交规范

建议使用 Conventional Commits：

```text
feat: add rag chat page
fix: resolve login token refresh issue
docs: update frontend architecture document
style: adjust greenhouse dashboard layout
refactor: split dashboard chart components
```

## 14. 总结

本前端系统以农业 AI 场景为核心，将 RAG 知识问答、知识库管理、农业数据大屏、专家工作台、基因智查、病害识别与温室监控整合到统一的 React 应用中。系统在交互层面强调清晰的模块入口、可追溯的 AI 回答来源、实时化的数据展示和面向农业生产的业务闭环；在技术层面基于 React 18、Vite、TypeScript、Ant Design 和 Zustand 构建轻量、可维护、可扩展的前端架构。

后续开发应围绕“模块清晰、数据可信、AI 可解释、体验统一”的原则推进：数据总览负责呈现平台态势，工作台承接日常任务，AI 对话与基因智查承载 RAG 核心能力，病害识别连接视觉 AI 应用，温室监控通过 3D 大棚和 IoT 数据呈现农业生产现场。各模块既能独立演进，也能通过统一的用户体系、知识库体系和接口规范形成完整平台能力。
